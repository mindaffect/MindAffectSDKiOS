Settings

  • Setting

    Objects conforming to this protocol can be used as Settings. This a.o. means they have a value that can be set and get, and that the NoiseTagging framework can provide the user with UI to edit the setting.

    The easiest way to conform to Setting is to subclass Setting_abstract.

    See also

    Settings can be grouped together in a SetOfSettings.
    See more

    Swift

    @objc
public protocol Setting : NSCopying, NSObjectProtocol

  • NoiseTagSettingTitles

    Struct containing titles of settings used by the NoiseTagging framework.

    The Noise Tagging framework makes use of settings, as defined by NoiseTagging.settings. You access these settings by their titles, which are defined in this struct.

    Example

    let setting_framesPerBit = NoiseTagging.settings.settingWith(title: NoiseTagSettingTitles.framesPerBit)

  • ColorSetting

    A Setting with a color value.

    The color is defined by one of two options:

    1. It is one of the ‘pickable colors’, which are predefined colors (PickableColor) as set on var pickableColors.
    2. It is specified using RGBA.
