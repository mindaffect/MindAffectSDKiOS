NoiseTagControlProperties

public class NoiseTagControlProperties : NSObject

Each NoiseTagControl has a noiseTagging property of type NoiseTagControlProperties. The purpose of this noiseTagging property is to store a number of things that are needed to make noise tagging work. It is also your main point of access to define how each control behaves as a brain-pressable button.

  • identifier

    The control’s identifier.

    Controls optionally have an identifier. This is amongst others used for logging purposes.

    Declaration

    Swift

    public var identifier: String?
  • customUtopiaObjectID

    The control’s utopia object ID.

    By default customUtopiaObjectID is nil and object IDs are assigned automatically. However, you may want to these IDs explicitly. Whenever a control is pressed, a ‘SELECT’ message is send to the MA Decoder, including the pressed control’s utopia object ID. The MA Decoder forwards this message this to all of its clients. This can be used to press a button on one device, but in response execute an action on another device.

    Declaration

    Swift

    public var customUtopiaObjectID: Int?
  • participatesInFlickering

    Whether the control flickers.

    By default this is true. If it is set to false, a control will no longer flicker, but it can still be pressed using touch (assuming a noise tagging action has been assigned to it).

    Declaration

    Swift

    public var participatesInFlickering: Bool { get set }
  • enabled

    Whether the control is enabled.

    If a control is not enabled, calling setNoiseTagAction simply does not have an effect.

    Whether a click sound should be played when the control is pressed.

  • addAction(timing:closure:)

    Add an action. All of a control’s actions are executed when the control is pressed, be it by brain or by touch.

    Important

    This should only be called as part of an update (see the Main Progamming Guide). Also note that at the start of each update, all actions are removed from their controls.
