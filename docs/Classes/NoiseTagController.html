NoiseTagController

public class NoiseTagController : NSObject, SetOfSettingsDelegate, NetworkingHelperDelegate

This class provides a singleton NoiseTagController, which is responsible for coordinating everything that has to do with noise tagging. Access the singleton instance using the convenience variable NoiseTagging.

Updates

  • updateControls(updateCode:)

    Perform an update.

    The current set of controls as well as the actions assigned to them can only be changed as part of an update block.

    In most cases you do not call this function directly. Instead you use push(view:forNoiseTaggingWithDelegate:), pop(), and update(view:forNoiseTaggingWithDelegate:). However, if you have multiple of these calls in a row – e.g. two pops at once –, we recommend that you group them together in one update, using this function.

    • example:
    // Combine two pops in one single update:
NoiseTagging.updateControls {
    NoiseTagging.pop()
    NoiseTagging.pop()
}

  • blockTrials(forDuration:)

    Use this function whenever starting the next noise tagging trial should be postponed untill a certain task has been finished.

    • Example: if one of your buttons is pressed and as a result a different screen is opened in an animated fashion, then you should call this function at the start of that animation, and call unblockTrials once the animation is finished, or pass a duration.

    Attention

    If no duration is passed, each call to block needs to be matched by a call to unblock. Otherwise no new trial will ever be started.

    Declaration

    Swift

    public func blockTrials(forDuration duration: TimeInterval? = nil)

    Parameters

    duration

    If nil, you are blocking trials untill you call unblockTrials. If this is non-nil, unblockTrials is called automatically after the passed duration.
  • unblockTrials()

    This function is the counterpart to blockTrials(forDuration:). You call blockTrials(forDuration:) to postpone starting the next noise tagging trial untill a certain task has been finished. If you do this without passing a duration, you must call unblockTrials once that task is finished.

    Attention

    Call this function after and only after having called blockTrials(forDuration:) without a duration. For everything to work correctly it is critical that the number of ‘blocks’ is balanced with the number of ‘unblocks’.
