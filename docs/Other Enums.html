  • ScalingUI

    On iOS normally you use auto-layout to make UI work for different screen sizes. However, when designing UI that can be controlled using noise tagging, there are two important limitations:

    1. Changing the dimensions of a control may negatively influence how easily the MindAffect Decoder recognises that the user is looking at it.
    2. Users cannot scroll simply by dragging a finger.

    Therefore for NoiseTagging apps sometimes it makes more sense to only support iPad, which provides more screen estate. An advantage of iPad is that all iPads have a similar screen ratio, so we can design our UI for one iPad model and simply scale everything to support other iPad models as well. This struct helps you do this.

    In principle this also works for iPhone, but because of large differences in screen ratio this might deliver poor results.

    Important

    ScalingUI currently only supports landscape orientations.
