NoiseTagDelegate

@objc
public protocol NoiseTagDelegate : AnyObject

The main responsibility of a NoiseTagDelegate is to prepare a unit for brain control by assigning actions to controls.

Assigning actions

  • startNoiseTagControlOn(noiseTaggingView:)

    When a unit becomes the current unit – because it has been pushed, or because another unit was popped –, or if NoiseTagging.update(view:forNoiseTaggingWithDelegate:) is called, NoiseTagging calls this function on the delegate.

    • Example:
    func startNoiseTagControlOn(noiseTaggingView: UIView) {
       // Add noise tagging actions:

       // For some types of controls you add actions directly:
       someNoiseTagButtonView.noiseTagging.addAction(timing: 0) {
           // This code is executed when the button is pressed:
           NoiseTagController.print("Hello")
       }

       // For some types of controls you add actions indirectly. E.g. a NoiseTagPopupButtonView's action always is the same and you only need to enable brain control on it:
       someNoiseTagPopupButtonView.noiseTagControlIsOn = true
   }

    Declaration

    Swift

    @objc
optional func startNoiseTagControlOn(noiseTaggingView: UIView)

Customization

  • customBackgroundColorFor(noiseTaggingView:)

    Optionally define an alternative background color for the view. By default the color is defined by NoiseTagController‘s defaultBackgroundColorPrimary setting.

  • idsOfControlsToIncludeAsTargetInVerifyMode(noiseTaggingView:)

    In some modes the framework randomly selects one of the controls as the target. By implementing this function, the delegate can limit the set of controls that can be selected as the target.
