Setting

@objc
public protocol Setting : NSCopying, NSObjectProtocol

Objects conforming to this protocol can be used as Settings. This a.o. means they have a value that can be set and get, and that the NoiseTagging framework can provide the user with UI to edit the setting.

The easiest way to conform to Setting is to subclass Setting_abstract.

See also

Settings can be grouped together in a SetOfSettings.

  • validityCheck

    A closure which checks whether a certain value is a valid value for the setting.

    This closure gets a ‘candidate’ for the setting’s new value and should return nil if that candidate is ‘valid’. If it is not valid, the closure should return an explanation for the user why the value is not valid.

    Attention

    Classes that implement the Setting protocol should perform the validityCheck before they actually change their setting value. They should also not forget to copy the validityCheck in their copy function.

    Declaration

    Swift
