Page

public class Page : NSObject

A Page defines content that can be displayed in a NavigatorTreeOfPages.

Pages are always used in conjunction with NavigatorTreeOfPages, which makes it easy to implement UI similar to what UINavigationController provides, but controllable via brain. A Page always has a title. Depending on the other properties you set, you use it in one of three possible ways:

  1. View Pages: If you set a view, when the page becomes NavigatorTreeOfPages‘s current page, NavigatorTreeOfPages simply shows that view beneath its navigation bar.
  2. Settings Pages: If you set an array of settings, the page will create a view, set it on itself, and populate it with UI for editing those settings. This means a Settings Page actually is a specific kind of View Page.
  3. List Pages: If you set subPages, when the page becomes NavigatorTreeOfPages’s current page, NavigatorTreeOfPages shows one row per subPage and lets the user navigate to the subPages.

Note that in the first two cases, it is still possible to push pages on NavigatorTreeOfPages, but in the third case pushing pages is handled by NavigatorTreeOfPages.

Note

Settings pages assume the orientation is landscape.
