ScalingUI

public enum ScalingUI

On iOS normally you use auto-layout to make UI work for different screen sizes. However, when designing UI that can be controlled using noise tagging, there are two important limitations:

  1. Changing the dimensions of a control may negatively influence how easily the MindAffect Decoder recognises that the user is looking at it.
  2. Users cannot scroll simply by dragging a finger.

Therefore for NoiseTagging apps sometimes it makes more sense to only support iPad, which provides more screen estate. An advantage of iPad is that all iPads have a similar screen ratio, so we can design our UI for one iPad model and simply scale everything to support other iPad models as well. This struct helps you do this.

In principle this also works for iPhone, but because of large differences in screen ratio this might deliver poor results.

Important

ScalingUI currently only supports landscape orientations.
  • scale(view:withScaleX:scaleY:scaleR:)

    When using NoiseTagging, you may want to use the same layout independent from the screen size (in points). E.g. if you have a layout defined for a regular iPad screen, if your app runs on an iPad with a larger screen, you might simply want to scale everything up. This is hard to do with autolayout, because that is better suited to fitting more content in if there is more space (which for most apps is what you want). This function makes it easy to scale everything up, but it only works for views in which no autolayout constraints are defined. Note that only view frames, layer frames, and fonts are scaled. Also note that there are convenience functions for going from a normal sized iPad or a 12.9-inch iPad to any other iPad.

    public static func scale(view: UIView, withScaleX scaleX: CGFloat, scaleY: CGFloat, scaleR scaleROrNil: CGFloat? = nil)

    Parameters

    view

    The view which needs to be scaled. All of the view’s contents are scaled as well.
    scaleX

    The amount of scaling applied horizontally.
    scaleY

    The amount of scaling applied vertically.
    scaleR

    The amount of scaling applied when the ratio needs to remain the same, such as for scaling text. If nil, the minimum of scaleX and scaleY is applied.
