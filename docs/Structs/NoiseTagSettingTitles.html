NoiseTagSettingTitles

public struct NoiseTagSettingTitles

Struct containing titles of settings used by the NoiseTagging framework.

The Noise Tagging framework makes use of settings, as defined by NoiseTagging.settings. You access these settings by their titles, which are defined in this struct.

Example

let setting_framesPerBit = NoiseTagging.settings.settingWith(title: NoiseTagSettingTitles.framesPerBit)

Flicker

  • useMetal

    Whether to use Metal instead of a display link for making the controls flicker.

    The NoiseTagging framework provides two ways to make the controls flicker. If this setting is false, it uses a CADisplayLink.

    Metal has the advantage of rendering the flicker more precise, so performance might increase by enabling this setting. However, there also is a downside: while your controls flicker, their appearance and frame should not change. All Metal drawing happens in a separate layer on top of your own UI and if you e.g. move your controls while flickering, the flicker of those controls – drawn in that separate layer -, will not move along.

    If your controls only change in between trials, we recommend you enable this setting.

Noise tags

  • framesPerBit

    Noise tags can be described as sequences of ones and zeros, where each 1 means white and 0 means black. These ones and zeros we call bits. This setting determines how many frames are used to display one bit. In other words: change this setting to change the speed by which noise tags are being played.

    See also

    preferredNFramesPerSecond

    Declaration

    Swift

    public static let framesPerBit: String
  • preferredNFramesPerSecond

    Preferred frame rate for playing noise tags.

    iOS can run at various frame rates. Some iPads by default run at 60Hz, while newer iPads by default dynamically switch between running at 60Hz and 120Hz. This setting determines the frame rate at which noise tags will be played.

    Declaration

    Swift

    public static let preferredNFramesPerSecond: String
  • filenameNoiseTags

    The name of the .txt file that defines the noise tags that should be used.

    Declaration

    Swift

    public static let filenameNoiseTags: String
  • assignNoiseTagsInReversedOrder

    Whether to assign noise tags to controls in reversed order.

    This is just for debugging. E.g. if you have a grid of control and this value is false, then by default the most top-left control gets the first noise tag and the most bottom-right control gets the last noise tag. If this setting is true, the order is reversed. This may be useful e.g. to research whether differences in performance between controls is due to the used noise tags or not.
