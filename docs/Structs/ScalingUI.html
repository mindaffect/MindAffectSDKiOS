ScalingUI

public struct ScalingUI

On iOS normally you use auto-layout to make UI work for different screen sizes. However, when designing UI that can be controlled using noise tagging, there are two important limitations:

  1. Changing the dimensions of a control may negatively influence how easily the MindAffect Decoder recognises that the user is looking at it.
  2. Users cannot scroll simply by dragging a finger.

Therefore for NoiseTagging apps sometimes it makes more sense to only support iPad, which provides more screen estate. An advantage of iPad is that all iPads ever released have the same screen ratio, so we can design our UI for one iPad model and simply scale everything to support other iPad models as well. This struct helps you do this.

  • scaleCompletely(view:withScale:)

    When using NoiseTagging, you may want to use the same layout independent from the screen size (in points). E.g. if you have a layout defined for a regular iPad screen, if your app runs on an iPad with a larger screen, you might simply want to scale everything up. This is hard to do with autolayout, because that is better suited to fitting more content in if there is more space (which for most apps is what you want). This function makes it easy to scale everything up, but it only works for views in which no autolayout constraints are defined. Note that only view frames, layer frames, and fonts are scaled. Also note that there is a convenience function for going from a normal sized iPad to any other iPad.
